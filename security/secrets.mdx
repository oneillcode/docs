## Creating Secrets

<Steps>
  <Step title="Navigate to Secrets">Go to Settings > Secrets</Step>
  <Step title="Add New Secret">Click "New Secret" and enter details</Step>
  <Step title="Set Permissions">Choose who can access the secret</Step>
  <Step title="Use in Applications">Reference secrets in your code</Step>
</Steps>

## Using Secrets

```javascript
// Reference a secret in your code
const apiKey = Secrets.get("STRIPE_API_KEY");

const response = await fetch("https://api.stripe.com/v1/charges", {
  headers: {
    Authorization: `Bearer ${apiKey}`,
  },
});
```

## Best Practices

<AccordionGroup>
  <Accordion title="Never Hardcode Secrets" icon="ban">
    Always use the secrets manager, never embed credentials in code
  </Accordion>
  <Accordion title="Rotate Regularly" icon="arrows-rotate">
    Implement regular secret rotation policies
  </Accordion>
  <Accordion title="Principle of Least Privilege" icon="user-lock">
    Grant minimal access required for each secret
  </Accordion>
  <Accordion title="Monitor Access" icon="eye">
    Review secret access logs regularly
  </Accordion>
</AccordionGroup>

## Environment-Specific Secrets

Maintain different secrets for each environment:

```javascript
// Development
const devApiKey = Secrets.get("API_KEY", "development");

// Production
const prodApiKey = Secrets.get("API_KEY", "production");
```

<Warning>
  Never use production secrets in development or staging environments.
</Warning>
